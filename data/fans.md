# fans.js 数据编辑说明

本文档说明 `data/fans.js` 中番种数据的完整格式，供协作者参考。

---

## 一、文件整体结构

`fans.js` 包含三个全局常量，按顺序定义：

| 常量 | 说明 | 是否需要编辑 |
|------|------|-------------|
| `FAN_TIERS` | 番数层级定义（88/64/48…/0） | 一般不动 |
| `ALL_TAGS` | 标签总表 | 新增标签时才编辑 |
| `FANS_DATA` | 番种数组，主体内容 | **主要编辑目标** |

---

## 二、FANS_DATA 条目字段

每个番种是 `FANS_DATA` 数组中的一个对象字面量。

### 必填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `id` | `string` | 全局唯一标识，英文小写+下划线（如 `da_si_xi`） |
| `name` | `string` | 番种主名称（如 `"大四喜"`） |
| `fan` | `number` | 番数值；`0` 代表立直 |
| `desc` | `string` | 一句话简介，显示在卡片正面 |
| `tags` | `string[]` | 标签数组，值必须已存在于 `ALL_TAGS` |
| `meldAllowed` | `boolean` | `true` = 允许副露；`false` = 必须门清 |
| `source` | `string` | 规则来源，如 `"国标"` `"日麻"` `"村规"` `"国标/日麻"` |
| `handTypes` | `string[]` | 适用牌型，取值见下表；空数组 `[]` = 不限 |

**`handTypes` 取值：**

| 值 | 含义 |
|----|------|
| `"33332"` | 标准牌型（4组面子+1将） |
| `"七对"` | 七对子型 |
| `"不靠"` | 七星不靠（全不靠）型 |
| `"国士"` | 国士无双型 |

### 选填字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `nameAlt` | `string[]` | 别名，如 `["国士无双"]` |
| `conditions` | `string` | 成立的附加条件，显示在展开详情中 |
| `excludes` | `string[]` | 同时不计的番种名（对应 `name` 字段的字符串） |
| `notes` | `string` | 规则补充说明（**黄色**引用框显示） |
| `tips` | `string` | 常见疑问/易错点（**蓝色**引用框显示） |
| `dlc` | `boolean` | `true` = DLC 番种，默认隐藏，需手动开启显示 |

### 牌型示例字段

| 字段 | 类型 | 说明 |
|------|------|------|
| `example` | `string` | **主例牌**，卡片收起时显示，写法见第三节 |
| `examples` | `object[]` | 展开后的多副例牌，写法见第四节 |

---

## 三、牌码对照表

`example` 和 `examples[].tiles` 字段使用以下牌码，以**空格**分隔。

### 数牌

|    | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 | 赤五 |
|--  |---|---|---|---|---|---|---|---|---|------|
| 万 | `1m` | `2m` | `3m` | `4m` | `5m` | `6m` | `7m` | `8m` | `9m` | `0m` |
| 饼 | `1p` | `2p` | `3p` | `4p` | `5p` | `6p` | `7p` | `8p` | `9p` | `0p` |
| 索 | `1s` | `2s` | `3s` | `4s` | `5s` | `6s` | `7s` | `8s` | `9s` | `0s` |

### 字牌

| 东 | 南 | 西 | 北 | 中 | 发 | 白 |
|----|----|----|----|----|----|----|
| `E` | `S` | `W` | `N` | `Z` | `F` | `B` |

### 特殊牌

| 代码 | 含义 | 用途 |
|------|------|------|
| `X` | 牌背 | 暗杠的首尾两张扣牌 |
| `front` | 空白牌面 | 暗杠中间两张正面牌，或示意用 |

### 分隔符与修饰符

| 符号 | 位置 | 含义 |
|------|------|------|
| `\|` | 单独token | 面子/副露之间的分隔线 |
| `>` | 牌码前缀 | 横置（顺时针旋转90°），用于标记从牌河捡入的那张牌 |

**横置示例：**

```
// 吃：从上家捡入 3m，成 3m 4m 5m
>3m 4m 5m

// 碰：从任意家捡入 E，成 E E E
E >E E

// 明杠：从任意家捡入 1p，成 1p 1p 1p 1p（通常最右横置）
1p 1p 1p >1p
```

> 横置牌在面子内的**位置**表示来源方向（右家→右端，对家→中间，左家→左端），暂不强制要求——填写时按实际或统一放右端均可。前端显示逻辑待后续实现。

## 四、examples 对象格式（完整版）

`examples` 数组中的每个对象支持以下字段：

```js
{
  label:  "说明文字",      // 可选，显示在牌图上方的小标题
  tiles:  "牌码字符串",    // 手牌，用 | 分隔面子，写法见下文
  agari:  "牌码",          // 可选：单张和牌（与手牌分开显示）
  wait:   ["牌码", "牌码"],// 可选：多张候选和牌（双碰/多面张等）
  tsumo:  true,            // 可选：true=自摸；省略或 false=荣和（默认）
  river:  "牌码字符串",    // 可选：牌河（按出牌顺序，建议 ≤ 20 张）
}
```

> `agari` 与 `wait` 互斥，同时只用一个：
> - 只有一张和牌 → 用 `agari`
> - 有两张或以上的候选和牌 → 用 `wait`

## 五、四种牌型展示格式

### 格式一：13+1（最常见）

**含义**：13 张手牌处于听牌状态，再摸/荣 1 张成牌。  
**写法**：`tiles` 写 13 张手牌，`agari` 写 1 张和牌。

```js
{
  label: "荣和",
  tiles: "1m 2m 3m | 4m 5m 6m | 7m 8m 9m | 1p 1p 1p | 2p",
  agari: "2p",
  tsumo: false,
}
```

> `tiles` 的最后一组只写 2 张将牌，`agari` 单独写第 3 张（此例为单吊将牌）。
> 若是两面/边张等听法，把完整的 13 张都写进 `tiles` 即可。

---

### 格式二：12+2（双碰/多面张）

**含义**：手牌听牌，有 2 张（或以上）不同的候选和牌（如双碰）。  
**写法**：`tiles` 写手牌，`wait` 列出所有候选和牌。

**双碰示例**（最后两组各只写 2 张，等待补全其中一组）：

```js
{
  label: "双碰待（1p 或 2p）",
  tiles: "1m 2m 3m | 4m 5m 6m | 7m 8m 9m | 1p 1p | 2p 2p",
  wait: ["1p", "2p"],
}
```

**九莲宝灯九面张示例**（9 种牌均可和）：

```js
{
  label: "纯正九莲（饼）",
  tiles: "1p 1p 1p 2p 3p 4p 5p 6p 7p 8p 9p 9p 9p",
  wait: ["1p", "2p", "3p", "4p", "5p", "6p", "7p", "8p", "9p"],
}
```

---

### 格式三：16+（大牌型，如四杠子）

**含义**：杠牌展开显示全部 4 张，总牌数超过 14 张。  
**写法**：`tiles` 按实际展示牌数写，无需强制 14 张。

暗杠写法：`X front front X`（首尾扣牌，中间两张正面）  
明杠写法：4 张正常牌码（如 `1m 1m 1m 1m`）

**四杠子示例**（18 张：4 组杠×4 + 2 将）：

```js
{
  label: "四暗杠",
  tiles: "X front front X | X front front X | X front front X | X front front X | Z Z",
  agari: "Z",
  tsumo: true,
}
```

```js
{
  label: "四明杠",
  tiles: "1m 1m 1m 1m | E E E E | 9p 9p 9p 9p | B B B B | Z Z",
  agari: "Z",
  tsumo: false,
}
```

---

### 格式四：显示牌河

**含义**：在手牌/和牌之外，同时展示牌河（流局、振听检查等场景）。  
**写法**：在 `tiles`/`agari` 基础上额外加 `river` 字段。

```js
{
  label: "荒牌流局",
  tiles: "1m 9m 1p 9p 1s 9s E S W N Z F B",
  river: "2m 3p 4s 5m E 6p 7s 8m N 9m 2p 3s 4m 5p 6s 7m 8p N 1p 2s",
}
```

> `river` 中的牌码按**出牌顺序**从左到右排列，与手牌同样使用空格分隔、`|` 可选分隔副露来源。建议不超过 20 张。

## 六、`example`（主例牌）的写法

`example` 字段是一个普通字符串，卡片**收起状态**下唯一显示的牌型行。

- 建议选最典型的牌型，张数适中（正常不超过 14 张，四杠子可至 18 张）
- 用 `|` 分组，便于阅读
- 无需标注和牌（收起状态不区分听牌/和牌，视觉展示即可）
- 若番种有多种完全不同的牌型变体，`example` 选代表性的一种，其余变体放入 `examples`

```js
// 好的写法
example: "1m 2m 3m | 4m 5m 6m | 7m 8m 9m | 1p 1p 1p | 2p 2p"

// 有 example 时，examples 展示细节变体
examples: [
  { label: "万子顺子", tiles: "1m 2m 3m | ...", agari: "2p" },
  { label: "含赤五", tiles: "1m 2m 3m | 4m 0m 6m | ...", agari: "2p" },
]
```

## 七、`tips` 与 `notes` 的区别

| 字段 | 颜色 | 面向对象 | 内容类型 |
|------|------|----------|----------|
| `tips` | 蓝色 | **玩家** | 常见疑问解答、易错点提示 |
| `notes` | 黄色 | **规则** | 规则补充、特殊裁定、村规差异 |

```js
notes: "（村）可额外计 64 番。",            // 规则补充
tips:  "将牌可以是任意牌，不限字牌或数牌。", // 玩家疑问
```

## 九、注意事项

1. **`id` 全局唯一**：同一 `id` 不能出现两次，命名用拼音全拼或首字母+下划线均可。
2. **`tags` 来源限制**：`tags` 数组中的每个值必须已存在于 `ALL_TAGS`；如需新增标签，先在 `ALL_TAGS` 中添加。
3. **`excludes` 写名称**：`excludes` 中填另一番种的 `name`（中文名），而非 `id`。
4. **`example` 优先**：每个条目都尽量填 `example`，这是卡片收起时唯一可见的视觉内容，缺失时卡片看起来较空洞。
5. **`examples` 补充变体**：若番种只有一种典型牌型，`examples` 可以省略；若有多种变体（不同花色/不同结构），用 `examples` 分别列出。
6. **`river` 字段目前仅存储数据**，前端显示逻辑待后续实现，先按格式填写即可。
